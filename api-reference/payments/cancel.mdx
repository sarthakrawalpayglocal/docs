---
title: "POST /gl/v1/payments/{ygTransactionId}/cancel"
description: "POST /gl/v1/payments/{ygTransactionId}/cancel — cancel a payment in INITIATED or PENDING status. Returns updated payment status on success."
---

Use this endpoint to cancel a payment that has not yet reached a terminal state. You can cancel a payment while the customer has not yet started checkout (`INITIATED`) or while the payment is being actively processed (`PENDING`). Once a payment reaches `SUCCESS`, `FAILED`, or `CANCELLED`, it can no longer be cancelled through this endpoint.

<Note>
You can only cancel payments with a status of `INITIATED` or `PENDING`. Attempting to cancel a payment in any other status will return a `422 Unprocessable Entity` error.
</Note>

## Endpoint

```
POST /gl/v1/payments/{ygTransactionId}/cancel
```

## Path parameters

<ParamField path="ygTransactionId" type="string" required>
  The PayGlocal transaction ID of the payment you want to cancel, as returned by the [initiate payment](/api-reference/payments/initiate) endpoint. Example: `YG_TXN_20240115_XYZ789`.
</ParamField>

## Request body

<ParamField body="reason" type="string" required>
  A human-readable reason for cancelling the payment. This is stored against the transaction for audit purposes and is visible in the PayGlocal merchant dashboard (e.g., `"Customer requested cancellation"`, `"Order was modified after payment was initiated"`).
</ParamField>

## Response fields

<ResponseField name="status" type="string" required>
  Status of the cancel API request. `"SUCCESS"` means the cancellation was accepted.
</ResponseField>

<ResponseField name="ygTransactionId" type="string" required>
  The PayGlocal transaction ID of the cancelled payment.
</ResponseField>

<ResponseField name="paymentStatus" type="string" required>
  The updated status of the payment after cancellation. This will be `"CANCELLED"` on a successful response.
</ResponseField>

## Code examples

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.sandbox.payglocal.in/gl/v1/payments/YG_TXN_20240115_XYZ789/cancel \
  --header "Authorization: Bearer YOUR_SANDBOX_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "reason": "Customer requested cancellation"
  }'
```

```javascript Node.js
const ygTransactionId = "YG_TXN_20240115_XYZ789";

const response = await fetch(
  `https://api.sandbox.payglocal.in/gl/v1/payments/${ygTransactionId}/cancel`,
  {
    method: "POST",
    headers: {
      Authorization: "Bearer YOUR_SANDBOX_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      reason: "Customer requested cancellation",
    }),
  }
);

const data = await response.json();
console.log(data.paymentStatus); // "CANCELLED"
```

```python Python
import requests

yg_transaction_id = "YG_TXN_20240115_XYZ789"

response = requests.post(
    f"https://api.sandbox.payglocal.in/gl/v1/payments/{yg_transaction_id}/cancel",
    headers={
        "Authorization": "Bearer YOUR_SANDBOX_API_KEY",
        "Content-Type": "application/json",
    },
    json={
        "reason": "Customer requested cancellation",
    },
)

data = response.json()
print(data["paymentStatus"])  # "CANCELLED"
```

</CodeGroup>

## Example response

```json
{
  "status": "SUCCESS",
  "ygTransactionId": "YG_TXN_20240115_XYZ789",
  "paymentStatus": "CANCELLED"
}
```

## Error: payment not cancellable

If you attempt to cancel a payment that is already in a terminal status (`SUCCESS`, `FAILED`, or already `CANCELLED`), the API returns a `422 Unprocessable Entity`:

```json
{
  "status": "FAILURE",
  "errorCode": "INVALID_PAYMENT_STATUS",
  "message": "Payment cannot be cancelled. Current status: SUCCESS"
}
```
