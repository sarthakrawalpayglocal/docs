---
title: "POST /gl/v1/payments/initiate"
description: "POST /gl/v1/payments/initiate — create a new payment session. Accepts order details and billing data, returns a redirectUrl for hosted checkout."
---

Use this endpoint to create a new payment transaction. You provide the order amount, currency, customer billing details, and the URL to redirect the customer back to your application after payment. PayGlocal returns a `redirectUrl` — send your customer to that URL to complete the payment on the PayGlocal hosted checkout page.

<Note>
Use `merchantUniqueId` as an idempotency key. If you send the same `merchantUniqueId` more than once, PayGlocal returns the original response instead of creating a duplicate transaction.
</Note>

## Endpoint

```
POST /gl/v1/payments/initiate
```

## Request body

<ParamField body="merchantId" type="string" required>
  Your merchant identifier as registered in the PayGlocal dashboard.
</ParamField>

<ParamField body="merchantTxnId" type="string" required>
  Your internal, unique transaction or order ID. This value is echoed back in the response and all subsequent status and webhook events so you can correlate PayGlocal transactions with your own records.
</ParamField>

<ParamField body="merchantUniqueId" type="string" required>
  Idempotency key for this request. If you retry a request with the same `merchantUniqueId`, PayGlocal returns the original response rather than creating a new transaction. Use a UUID or a similarly collision-resistant value.
</ParamField>

<ParamField body="paymentData" type="object" required>
  Object containing the payment amount, currency, and customer billing information.

  <Expandable title="paymentData properties">
    <ParamField body="paymentData.totalAmount" type="string" required>
      The total amount to charge, expressed as a string with two decimal places (e.g., `"1500.00"`). Amounts must be greater than zero.
    </ParamField>

    <ParamField body="paymentData.txnCurrency" type="string" required>
      ISO 4217 three-letter currency code for the transaction (e.g., `"INR"`, `"USD"`, `"EUR"`).
    </ParamField>

    <ParamField body="paymentData.billingData" type="object" required>
      Customer billing details used for payment processing and fraud checks.

      <Expandable title="billingData properties">
        <ParamField body="paymentData.billingData.firstName" type="string" required>
          Customer's first name.
        </ParamField>

        <ParamField body="paymentData.billingData.lastName" type="string" required>
          Customer's last name.
        </ParamField>

        <ParamField body="paymentData.billingData.emailId" type="string" required>
          Customer's email address. Used for payment confirmations and fraud screening.
        </ParamField>

        <ParamField body="paymentData.billingData.phoneNumber" type="string">
          Customer's phone number in international format (e.g., `"+919876543210"`). Optional but recommended for UPI and wallet payment methods.
        </ParamField>

        <ParamField body="paymentData.billingData.addressStreet1" type="string" required>
          First line of the customer's billing address.
        </ParamField>

        <ParamField body="paymentData.billingData.city" type="string" required>
          City of the customer's billing address.
        </ParamField>

        <ParamField body="paymentData.billingData.state" type="string" required>
          State or province code (e.g., `"MH"` for Maharashtra, `"CA"` for California).
        </ParamField>

        <ParamField body="paymentData.billingData.country" type="string" required>
          ISO 3166-1 alpha-2 country code (e.g., `"IN"`, `"US"`, `"GB"`).
        </ParamField>

        <ParamField body="paymentData.billingData.postalCode" type="string" required>
          Postal or ZIP code of the customer's billing address.
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="pgData" type="object" required>
  Payment gateway configuration for this transaction.

  <Expandable title="pgData properties">
    <ParamField body="pgData.returnUrl" type="string" required>
      The URL PayGlocal redirects the customer to after they complete (or abandon) the payment on the hosted checkout page. Include a transaction reference in the URL so you can identify the session on return (e.g., `"https://yourstore.com/order/confirm?txnId=ORD_001"`).
    </ParamField>

    <ParamField body="pgData.notifyUrl" type="string">
      An optional webhook URL specific to this transaction. If provided, PayGlocal posts payment status events to this URL in addition to any webhook URL configured in your dashboard.
    </ParamField>
  </Expandable>
</ParamField>

## Response fields

<ResponseField name="status" type="string" required>
  Overall status of the API request. `"SUCCESS"` indicates the payment session was created successfully. `"FAILURE"` indicates a problem with the request.
</ResponseField>

<ResponseField name="ygTransactionId" type="string" required>
  PayGlocal's globally unique identifier for this transaction. Store this value — you need it to check payment status, cancel the payment, or create a refund later.
</ResponseField>

<ResponseField name="merchantTxnId" type="string" required>
  Your transaction ID echoed back from the request for easy correlation.
</ResponseField>

<ResponseField name="redirectUrl" type="string" required>
  The URL you must redirect your customer to in order to complete the payment on the PayGlocal hosted checkout. This URL is time-limited; redirect the customer promptly after receiving it.
</ResponseField>
