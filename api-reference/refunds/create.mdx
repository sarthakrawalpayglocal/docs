---
title: "POST /gl/v1/refunds"
description: "POST /gl/v1/refunds — create a full or partial refund for a successful payment. Accepts the original transaction ID, amount, and idempotency key."
---

Use this endpoint to refund a customer for a payment that has reached `SUCCESS` status. You can issue a full refund for the original amount or a partial refund for a lesser amount. To prevent duplicate refunds, include a `merchantRefundId` — if you retry the request with the same value, PayGlocal returns the original refund record instead of creating a new one.

<Note>
Refunds can only be created for payments with a status of `SUCCESS`. Attempting to refund a payment in any other status will return a `409 Conflict` error.
</Note>

<Tip>
Partial refunds are supported. You can issue multiple partial refunds against the same `ygTransactionId` as long as the cumulative refunded amount does not exceed the original payment amount.
</Tip>

## Endpoint

```
POST /gl/v1/refunds
```

## Request body

<ParamField body="ygTransactionId" type="string" required>
  The PayGlocal transaction ID of the original successful payment you want to refund. This was returned in the `ygTransactionId` field of the [initiate payment](/api-reference/payments/initiate) response.
</ParamField>

<ParamField body="refundAmount" type="string" required>
  The amount to refund, expressed as a string with two decimal places (e.g., `"750.00"`). Must be greater than zero and must not exceed the original payment amount minus any previously refunded amounts.
</ParamField>

<ParamField body="refundCurrency" type="string" required>
  ISO 4217 currency code for the refund. This must exactly match the currency used in the original payment transaction.
</ParamField>

<ParamField body="refundReason" type="string" required>
  A human-readable reason for the refund, stored for audit and reporting purposes (e.g., `"Item out of stock"`, `"Customer return"`, `"Duplicate charge"`).
</ParamField>

<ParamField body="merchantRefundId" type="string" required>
  Your unique identifier for this refund request, used as an idempotency key. If you send a request with the same `merchantRefundId` more than once, PayGlocal returns the existing refund record rather than creating a duplicate. Use a UUID or a similarly collision-resistant value.
</ParamField>

## Response fields

<ResponseField name="refundId" type="string" required>
  PayGlocal's unique identifier for the refund. Store this value to check refund status later using the [GET /gl/v1/refunds/{refundId}](/api-reference/refunds/status) endpoint.
</ResponseField>

<ResponseField name="ygTransactionId" type="string" required>
  The PayGlocal transaction ID of the original payment.
</ResponseField>

<ResponseField name="merchantRefundId" type="string" required>
  Your refund ID echoed back from the request.
</ResponseField>

<ResponseField name="refundAmount" type="string" required>
  The refund amount as a string with two decimal places.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the refund. Possible values:

  | Value       | Description                                          |
  | ----------- | ---------------------------------------------------- |
  | `PENDING`   | Refund accepted and queued for processing.           |
  | `PROCESSED` | Refund successfully sent to the customer's account.  |
  | `FAILED`    | Refund processing failed; contact PayGlocal support. |
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp indicating when the refund was created (e.g., `"2024-01-16T09:00:00Z"`).
</ResponseField>

## Code examples

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.sandbox.payglocal.in/gl/v1/refunds \
  --header "Authorization: Bearer YOUR_SANDBOX_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "ygTransactionId": "YG_TXN_20240115_XYZ789",
    "refundAmount": "750.00",
    "refundCurrency": "INR",
    "refundReason": "Customer return — item defective",
    "merchantRefundId": "REF_20240116_001"
  }'
```

```javascript Node.js
const response = await fetch(
  "https://api.sandbox.payglocal.in/gl/v1/refunds",
  {
    method: "POST",
    headers: {
      Authorization: "Bearer YOUR_SANDBOX_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      ygTransactionId: "YG_TXN_20240115_XYZ789",
      refundAmount: "750.00",
      refundCurrency: "INR",
      refundReason: "Customer return — item defective",
      merchantRefundId: "REF_20240116_001",
    }),
  }
);

const data = await response.json();
console.log(data.refundId);  // e.g. "YG_REF_20240116_ABC123"
console.log(data.status);    // "PENDING"
```

```python Python
import requests

response = requests.post(
    "https://api.sandbox.payglocal.in/gl/v1/refunds",
    headers={
        "Authorization": "Bearer YOUR_SANDBOX_API_KEY",
        "Content-Type": "application/json",
    },
    json={
        "ygTransactionId": "YG_TXN_20240115_XYZ789",
        "refundAmount": "750.00",
        "refundCurrency": "INR",
        "refundReason": "Customer return — item defective",
        "merchantRefundId": "REF_20240116_001",
    },
)

data = response.json()
print(data["refundId"])  # e.g. "YG_REF_20240116_ABC123"
print(data["status"])    # "PENDING"
```
