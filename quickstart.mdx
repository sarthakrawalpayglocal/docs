This guide takes you through the minimum steps needed to initiate a payment and verify its status using the PayGlocal API. By the end, you'll have made a real API call against the sandbox and received a payment response you can inspect and build on.

<Steps>
  <Step title="Get your API credentials">
    Log in to the [PayGlocal merchant dashboard](https://dashboard.payglocal.in) and navigate to **Settings → API Keys**. You'll find separate key pairs for sandbox and production.

    Copy your sandbox secret key — it looks like this:

    ```
    sk_test_4eC39HqLyjWDarjtT7fin9oLpjD2HFmD
    ```

    <Tip>
      Keep your secret key out of your source code. Store it in an environment variable such as `PAYGLOCAL_API_KEY` and read it at runtime.
    </Tip>

    You'll also need your **Merchant ID**, visible in the dashboard under **Settings → Account**. It follows the format `MERCH_XXXXXX`.
  </Step>

  <Step title="Initiate a test payment">

  <Step title="Check payment status">
    After the customer completes (or abandons) the payment flow, PayGlocal redirects them to your `returnUrl`. At that point, call the status endpoint to get the current state of the payment.

    <CodeGroup>

    ```bash cURL
    curl --request GET \
      --url https://api.sandbox.payglocal.in/gl/v1/payments/PG_PAY_20240115_7f3a9b2c/status \
      --header 'Authorization: Bearer sk_test_4eC39HqLyjWDarjtT7fin9oLpjD2HFmD'
    ```

    ```javascript JavaScript
    const paymentId = 'PG_PAY_20240115_7f3a9b2c';

    const response = await fetch(
      `https://api.sandbox.payglocal.in/gl/v1/payments/${paymentId}/status`,
      {
        headers: {
          Authorization: 'Bearer sk_test_4eC39HqLyjWDarjtT7fin9oLpjD2HFmD',
        },
      }
    );

    const result = await response.json();
    console.log(result.data.status);
    ```

    ```python Python
    import requests

    payment_id = "PG_PAY_20240115_7f3a9b2c"

    response = requests.get(
        f"https://api.sandbox.payglocal.in/gl/v1/payments/{payment_id}/status",
        headers={
            "Authorization": "Bearer sk_test_4eC39HqLyjWDarjtT7fin9oLpjD2HFmD",
        },
    )

    result = response.json()
    print(result["data"]["status"])
    ```

    </CodeGroup>

  </Step>

  <Step title="Handle the response">
    Use the `data.status` field to determine the outcome and take action in your system:

    | Status | Meaning | Action |
    |---|---|---|
    | `INITIATED` | Payment created, customer not yet redirected | Wait for customer to proceed |
    | `PENDING` | Customer is on the payment page | Poll status or wait for webhook |
    | `PAID` | Payment completed successfully | Fulfil the order |
    | `FAILED` | Payment attempt was unsuccessful | Prompt the customer to retry |
    | `CANCELLED` | Payment was cancelled | Release reserved inventory |
    | `REFUNDED` | Full refund has been issued | Update your records |

    <Note>
      For production integrations, rely on [webhooks](/concepts/webhooks) rather than polling the status endpoint. Webhooks deliver real-time payment events to your server without requiring repeated API calls.
    </Note>
  </Step>
</Steps>
